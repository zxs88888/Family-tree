# UI/UX 设计文档（V1.6）：家族族谱管理 Web 应用

**版本**：V1.6 最终版
**匹配 PRD**：V1.8
**匹配 TDD**：V1.7

---

## 设计原则

1. **庄重与温情**：传统中国风色调，营造家族传承的仪式感
2. **触手可及**：交互目标最小 44x44 pt，文字不小于 16px
3. **极简主义**：页面永远只展示一个核心操作
4. **血脉可视化**："我"是图谱的视觉中心，金色连线代表直系血脉

## 2. 首页

### 2.1 顶部导航栏

```
[家族名称]                    [Aa] [👤] [⚙️] [↻]
```

- `Aa`：大字号模式切换（16px ↔ 20px），图标高亮 + Toast 反馈
- `👤`：切换身份，清空缓存，重新弹出"选我"弹窗
- `⚙️`：管理员入口（仅 `isAdmin` 可见）
- `↻`：退出/重置，清空 `my_member_id`、`access_code_verified`、eventsCache，调用 `supabase.auth.signOut()`，页面刷新（见 PRD 5.1.5）

### 2.2 图谱区域

- ECharts Graph 关系图，占屏幕 90%
- 左上角视图切换器浮层

### 2.3 视图切换器

```
┌─────────────────────────────┐
│ 🧬 脉系视图 │ 🌳 全局视图 │
└─────────────────────────────┘
```

- 选中态：`#8B1A1A` 红色背景 + 白色文字
- 未选中态：透明背景 + `#6B6B6B` 灰色文字
- 禁用态（`myMemberId === null`）：`opacity: 0.5`，点击 Toast 提示
- 共用数据源，切换仅改变 `centerId` 和 `depth`

### 2.4 少量成员图谱

成员 < 4 人时，图谱下方显示引导提示："家族成员较少，继续添加以展现更完整的家族脉络"，管理员可见"添加成员"快捷入口。

### 2.5 视图切换超时反馈

```html
<view v-if="switching" class="switch-loading">
  <view class="loading-spinner"></view>
  <text class="loading-text">
    {{ loadingTime > 3000 ? '家族成员较多，正在整理图谱...' : '加载中...' }}
  </text>
</view>
```

## 3. 访问码验证（AccessCodeModal）

### 3.1 布局

- 全屏居中弹窗
- 背景：家族姓氏书法字水印
- 输入框默认密文（`type="password"`）
- 右侧眼睛图标切换明文/密文（点击区域 ≥ 48x48）
- 自动获取焦点

### 3.2 动态提示

```html
<text class="default-hint">提示：访问码为 {{ accessCodeHint }}（联系管理员获取完整码）</text>
```

通过 RPC `get_access_code_hint` 获取脱敏提示。

### 3.3 键盘适配

```javascript
uni.onKeyboardHeightChange((res) => {
  const keyboardHeight = res.height || 0
  if (keyboardHeight > 0) {
    uni.pageScrollTo({ selector: '.access-code-input-wrapper', offsetTop: -100, duration: 200 })
  }
})
// 页面卸载时 uni.offKeyboardHeightChange()
```

### 3.4 验证状态

- 验证中：按钮禁用 + "验证中..."
- 验证失败：红色边框 + 红色错误提示
- 连续输错：3 次后提示"请确认后重试"，5 次后锁定 30 秒
- 验证成功：Toast "验证成功，正在加载家族数据..." → 匿名登录 → 1.5s 后关闭弹窗
- 匿名登录失败：Toast 错误提示，不关闭弹窗，允许重试

## 4. "选我"身份确认（OnboardingModal）

### 4.1 搜索

- 输入框带 🔍 图标
- 占位文字："输入您的姓名，在家族中找到自己"
- 实时模糊搜索，Debounce 300ms

### 4.2 结果列表

- 头像缩略图 + 姓名 + 父辈辅助信息，如"张伟（父亲：张建国）"
- 无结果时："未找到该成员，请检查姓名或联系管理员添加"
- 选中结果：金色边框 + 浅金色背景

### 4.3 确认

- 底部固定大按钮"✅ 确认这是我"（红色），仅在选中时可点击
- 确认后刷新图谱

### 4.4 跳过

```html
<button class="btn-skip" @click="skipOnboarding">稍后设置</button>
```

Toast 提示："您可以在右上角「👤」随时设置身份"

## 6. MemberDrawer（成员详情抽屉）

### 6.1 结构

- 顶部拖拽把手：灰色短横线 ——（宽 40px，高 4px，圆角 2px）
- 上滑展开至 80%，下滑恢复至 40%

### 6.2 折叠态（40%）

- 圆形头像（直径 60px）+ 姓名（22px，粗体）
- 性别 + 生卒年 / "🟢 在世"标签
- 金色分割线（40px 宽，居中）
- 生平简介（最多 3 行，超出显示"...展开"）

### 6.3 头像占位

无头像时：姓名首字母 + `#8B1A1A` 红色背景圆，白色文字 28px。

### 6.4 展开态（80%）

- 完整生平简介
- 关联关系列表：父亲、母亲、配偶、子女
- 关系无数据时显示"暂无"（`#AAAAAA`，斜体）

### 6.5 底部动作栏

```html
<button class="btn-set-identity" @click="setAsMe">设为我的身份</button>
<button v-if="isAdmin" class="btn-edit" @click="openEditor">编辑资料</button>
<text v-else class="hint-text">只读模式，如需修改请联系管理员</text>
```

**"设为我的身份"** 执行：
- 更新 `userStore.myMemberId`
- 清空事件缓存
- 刷新图谱
- Toast "已设为'xxx'的身份"
- 关闭抽屉

### 7.3 成员列表

- 展示姓名、性别、在世状态/生卒年
- 右侧操作：编辑（✏️）、软删除（🗑️）

- 表单字段：姓名、性别（单选）、生年、卒年（在世时隐藏）、是否在世开关、头像上传

|------|------|

| 上传后预览 | 表格预览前 10 行 + 时间线解析预览 |
| 导入中 | 进度条 + 不可操作 |

| 导入失败 | 红色 Toast + 错误列表 + "请修正后重新上传" |

- 默认折叠：年份 + 图标 + 标题 + "展开 ▾"

- 无图事件：图片区域留空，不显示任何提示

### 8.2 缩略图

- 展开后显示前 3 张（横向 80x80），超过 3 张显示 "+N"
- 折叠态不显示缩略图
- 点击缩略图 → 全屏预览

### 8.3 图库

- 入口：时间线底部"查看全部 X 张照片"按钮
- MVP 方案：`uni.previewImage` + `indicator: 'number'`
- 长按支持保存图片
- 自定义上下文浮层（年份+标题）作为 P2
- 空图库："这位成员暂无历史照片，欢迎补充。"

## 10. 动效与反馈

| 交互 | 动效 | 实现方式 |
|------|------|----------|
| 页面切换 | slideInUp 300ms ease-out | `transform: translateY(100%) → 0` |
| 抽屉弹出 | translateY 250ms | 绑定 touch 事件，跟随手势 |
| 节点点击 | 涟漪扩散 | `::after` 伪元素 + `@keyframes ripple { to { transform: scale(4); opacity: 0; } }` |
| 骨架屏 | 流光闪烁 1.5s | `@keyframes shimmer` |
| 按钮点击 | scale(0.96) | active 状态 |
| "我"节点呼吸 | shadowBlur 循环 | `requestAnimationFrame` + 帧率控制 |
| 直系连线发光 | 金色 + 阴影 | 与呼吸同步 |
| 视图切换 | 淡入淡出 200ms | `opacity` 过渡 |

### 10.1 Toast 反馈

| 类型 | 背景色 | 图标 | 停留 |
|------|--------|------|------|
| 成功 | `#27AE60` | ✓ | 2 秒 |
| 失败 | `#E74C3C` | ✗ | 3 秒 |
| 加载中 | `#3498DB` | ⟳ | 手动关闭 |

## 11. UI/UX 验收清单

- [ ] 金色血脉连续性：从"我"到始祖到末代，所有直系连线金色，无断裂
- [ ] 金色血脉与渲染深度一致：未渲染直系显示金色箭头或"…"提示
- [ ] 视图切换：脉系/全局切换流畅，共用数据源，渲染正确
- [ ] "我"节点发光：平滑呼吸光晕，CPU 低占用
- [ ] 节点位置稳定：刷新后不发生显著漂移
- [ ] 大字号切换：图标高亮 + Toast 反馈，节点和力导向参数自适应，长标签换行
- [ ] 抽屉拖拽：上滑 80%，下滑 40%
- [ ] 只读模式权限：隐藏"编辑资料"按钮，显示"只读模式"提示
- [ ] 访问码密文：默认圆点，眼睛切换明文（点击区域 ≥ 48px）
- [ ] 访问码动态提示：与数据库一致（脱敏显示）
- [ ] 连续输错锁定：5 次错误后锁定 30 秒
- [ ] 验证成功过渡：Toast 1.5s 后关闭弹窗，匿名登录失败有错误提示
- [ ] 空状态分场景：管理员显示添加按钮，只读用户显示联系方式
- [ ] Toast 反馈：成功 2 秒，失败 3 秒
- [ ] 事件标题截断：最多 2 行，超出省略号
- [ ] 图库预览：MVP 使用 uni.previewImage + indicator
- [ ] 键盘适配：输入框上移，不被键盘遮挡
- [ ] 头像占位：姓名首字母 + 红色背景圆
- [ ] 少量成员图谱：< 4 人时显示引导提示
- [ ] 无关系状态：显示"暂无"灰色斜体
- [ ] 切换身份：清空缓存，重新弹出"选我"
- [ ] 退出重置：清空缓存 + session，刷新回登录页
