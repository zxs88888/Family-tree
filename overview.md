## 后续追加修复（点击选中失效）

- **问题反馈**：用户反映「越改越差了，之前还能点击选中，现在点都不能点了」。
- **根因**：`FamilyGraph.vue` 的 `onPointerDown` 中直接 `setPointerCapture` 到 SVG，把指针事件捕获走，导致节点 `<g>` 上的 `@click.stop` 被吞/重定向到背景，点击永远选不中。
- **修复**：
  - 按下只记录起点，不立即捕获；
  - 超过 `4px` 移动阈值才判定为拖拽并捕获指针；
  - 节点增加透明命中圆（覆盖名卡/称谓），点名卡也能选中；
  - 拖拽结束用 `justPanned` 抑制残影 click，避免拖拽松开后误触发取消选中；
  - 背景点击仅在真正的 `click` 事件上取消选中，节点点击通过 `stop` 阻止冒泡。
- **验证**：Puppeteer 真实模拟点击 17 项断言全通过。截图显示点「我」时姐姐可见不淡、姐夫/外甥女淡化；点「姐姐」时姐姐家系（含姐夫/外甥女）高亮、我作为同父同母兄弟可见不淡。
- **提交**：`fix(graph): SVG 族谱节点点击选中失效`（`615000d`）已 push 到 `origin/main`。

## 选父母时金线连到孙辈（外甥女）

- **反馈**：用户发现「当选择父母时，外甥女角色是亮着的，但金色脉络没有亮」。
- **根因**：`src/utils/elbowGraph.js` 的 `computeH` 只把「选中者本人」的配偶纳入 H，但后代的垂落支连线端点含其配偶父母（外甥女 yt 的父亲是姐夫 xy）。yt 已因后代 BFS 进入 H，但 xy 不在 H，导致 `nodes=[yt, xy, xn]` 的垂落支无法成为金线。
- **修改（`src/utils/elbowGraph.js`）**：把 H 的配偶计算从「仅选中者」扩展为「所有血脉成员的配偶」，保证金线从选中者到子孙全程连续。
- **影响验证**：
  - 选父母 t：金色脉络从父母经子女、姐夫连到外甥女。
  - 选我(xs)：我无配偶，父母配偶互在 H，姐夫 xy / 外甥女 yt 仍被排除，仍淡出 —— 不影响此前规则。
  - 选姐姐 xn：姐夫/外甥女已在姐姐脉络内，规则保持一致。
- **验证**：Puppeteer 点击断言 23/23 通过；新增选父母 t → 外甥女 yt 有金线断言；`npm run build:h5` 通过。
